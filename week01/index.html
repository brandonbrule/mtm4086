Git Add Remote Repo

Simple Command Line Reminder to Stay Up-To-Date With Other Repositories

Let's say I want to work with another repository and keep those files up-to-date in my own repo.

WordPress or Drupal is a great example of working with remote repositories.

We can merge in core updates as they are released and maintain a safe backup of the previous state in doing so. Git is amazeballs.

Dun du du daa... git remote repos

I will be using github for my hosted repository.

I start with cloning my github repository. This is where my entire site will live.

git clone

git clone https://github.com/brandonbrule/my-repository.git

Make sure to go into the newly created repository directory.

cd my-repository

Next, we'll add WordPress as a remote repository to our own repository.

git remote add [name] [protocol://path.git]

git remote add wordpress https://github.com/WordPress/WordPress.git

A quick test to list all the repositories

git remote -v

This will list my repository as the origin and the WordPress repository as wordpress.

Remember, it's git remote add [name I've set] [path]

$git remote -v
origin https://github.com/brandonbrule/my-repository.git (fetch)
origin https://github.com/brandonbrule/my-repository.git (push)
wordpress      https://github.com/WordPress/WordPress.git (fetch)
wordpress      https://github.com/WordPress/WordPress.git (push)

Okay Awesome. The repos are listed, but there's no WordPress files on my computer yet.

git fetch

git fetch wordpress

This will do its git thing, and come back with a list of branches and tags from the WordPress repository.

Something like this will be returned.

* [new branch]    3.5-branch -> wordpress/3.5-branch
* [new branch]    3.6-branch -> wordpress/3.6-branch 
* [new branch]    3.7-branch -> wordpress/3.7-branch
* [new branch]    3.6-branch -> wordpress/3.6-branch
* [new branch]    master     -> wordpress/master from https://github.com/WordPress/WordPress
* [new tag]       1.5.1.2    -> 1.5.1.2
* [new tag]       2.0.1      -> 2.0.1
* [new tag]       2.5        -> 2.5

Now we'll pull in the files of the master branch.

pull = get files, push = transfer my changes

git pull wordpress master

And downloading of the files will begin.

Congratulations, you've successfully added a remote repository into your own.

Now the usual steps of git add, commit and happy fun times here we come. Everytime I want to pull in the newest version of WordPress I can just type

git pull wordpress master

and I will get those changes merged into my site repo, no worries about updates - revert back anytime.

git add .

git commit -m "I'm committing WordPress. Horray"

git push origin master

git add .(all)
